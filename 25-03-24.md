## 🗂️ Topics Covered

### 1. 🎨 Terminal Color Schemes & Customization
The lecture opened with a deep dive into terminal theming. The instructor walked through:
- The difference between **8-color**, **16-color**, **256-color**, and **true color** (hex code) support in terminals
- How each color slot (e.g. "red") is configurable — so if you remap your red to green, programs that print "red" text will show green instead
- The website **terminal.sexy** as a tool for previewing and building color schemes
- Popular ready-made themes (e.g. **Nord**, **Solarized**, **Gruvbox**, **Catppuccin**) and why popular themes are useful — they tend to be available across many tools (editors, terminals, IDEs) consistently

### 2. 🗃️ Git — Version Control Basics
A large portion of the session was dedicated to **Git** fundamentals:
- **`git init`** — initialising a repository; the hidden `.git` folder stores all history
- **`git status`** — checking what's going on in your repo (the instructor described this as "very important and informative")
- **`git add`** — staging changes (the "snapshot" analogy was used)
- **`git commit`** — saving the staged snapshot with a message
- Removing `.git` with `rm -rf .git` to un-initialise a repo
- **`git branch -M main`** — renaming the default branch to `main`
- **`git remote add origin`** — linking the local repo to a remote (GitHub)
- **`git push -u origin main`** — pushing code to GitHub for the first time; logging in via browser

### 3. 📝 Markdown
Towards the end of the session, the instructor introduced **Markdown** as a lightweight text formatting language used widely in development (README files, blogs, documentation):
- Headings (`#`, `##`, `###`)
- Bold, italic, strikethrough text
- Lists (ordered and unordered)
- Links and images
- Code blocks (inline and fenced), with syntax highlighting via language tags
- Tables (noted as "a bit painful to write by hand" — plugins recommended)
- Markdown is used far beyond code — e.g. blog platforms, chat apps, GitHub, and more
